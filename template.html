We start with JIRA

Every feature development starts with an JIRA story.

From an JIRA story you create an feature branch by clicking on the link. (see image)

This window appears on the right when clicking on the story link from the board (work mode)

Create feature branch in Stash

The name of the feature branch will be the same as the JIRA story

Check out branch via Stash or SourceTree

If you created the feature branch, than checkout the branch with the commandline or SourceTree.

And start developing!

Finished developing?

Merge the master in to your feature branch.

Commit an push all the changes to your feature branch.

Do an pull-request in Stash.

Click the button "pull request".

Fill in the pull request.

Fill in the form pull-request.

The most things will be automaticly filled in.

Just mention an other developer and tester to get approval for pull request.

Did someone mention you?

If your an tester?
Than checkout the feature branch on your localmachine and test the application.

If your an developer?
Peer-review the code, add comments as needed and checkout the feature branch on your localmachine and test the application.

If the application works and the code is valid, click the button "approve" in the top right corner. Then click on the button "merge" to merge the feature branch into the master branch.

Is the pull-request approved?

If your the developer of the branch?
Go to JIRA and un-assign the task. Drag the task to verify, and assign it to the tester.

If your an tester?
If you approved the pull-request, than drag the JIRA task to done. (to keep the JIRA board up-to-date).
